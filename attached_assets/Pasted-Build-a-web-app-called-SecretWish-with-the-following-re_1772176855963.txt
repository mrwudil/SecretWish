Build a web app called SecretWish with the following requirements:

1. **Landing Page (Sender Entry)**
   - Headline: "Stop Guessing Her Birthday Gift" (bold, large, centered, #1E1E2F)
   - Subtext: "She picks 3 things. You choose 1. She never knows you asked." (#555555)
   - CTA Button: "Create Secret Wish" (rounded edges 8px, #FF6B6B, hover scale 1.05x, subtle shadow)
   - Background: Gradient #F7F7F7 → #E0E0F7
   - Optional small floating gift box illustration
   - Clicking CTA navigates to Sign Up / Auth page

2. **Sign Up / Auth Screen**
   - Sign in with Google
   - Email login / signup
   - Minimalist card layout, rounded corners, shadow, white background

3. **Create Secret Wish Page (Sender)**
   - Input fields:
     - Receiver Email (validate format)
     - Event Type: Birthday (default, disabled)
     - Reveal Identity: "After Purchase" or "Never" (radio buttons)
   - CTA Button: "Send Secret Wish" (#FF6B6B)
   - Style: Centered card, soft shadow, rounded edges
   - On submit:
     - Generate unique link (secretwish.com/r/ABC123)
     - Store question in Supabase DB (tables: users, questions, wishes, statuses)
     - Send anonymous email via transactional email (SendGrid / Postmark)
     - Link in email points to Receiver Wish Selection page

4. **Receiver Wish Selection Page**
   - Headline: "If someone could grant you 1 birthday wish… what would it be?" (#1E1E2F)
   - Instructions: "Add up to 3 things that would make you happy. Keep it realistic 👀" (#555555)
   - Wish Input Fields (3 max):
     - Item Name (text)
     - Optional Link (URL)
     - Optional Price (number)
   - Add Item Button: "+" (max 3)
   - Submit Button: "Submit Wishes" (#FF6B6B)
   - On submit:
     - Validate max 3 items
     - Store in Supabase DB
     - Show confirmation: "Your wishes have been sent anonymously. Someone is reviewing them 👀"

5. **Sender Dashboard**
   - Header: Logo + greeting "Hi [Sender Name]"
   - Display wish cards (1-3 per receiver):
     - Item Name
     - Optional Link
     - Price
   - Action Buttons per card:
     - "Mark as Surprise in Progress" (#FF6B6B) → updates status in DB, triggers notification
     - "Not this time" (#A0A0A0) → updates status, soft tone notification
   - Card style: Rounded corners 12px, soft shadow, white background
   - Layout: Grid or list, mobile responsive

6. **Email Notifications**
   - To Receiver: "🎁 Someone wants to know your birthday wish. Tap the link: [SecretWish Link]" (from hello@secretwish.com)
   - To Sender: "Your secret question has been answered. View wishes: [Link]"
   - Mobile-friendly formatting

7. **Repeat Engagement / Future Hooks**
   - Banners for future events: "Anniversary Mode coming soon! Just Because Surprises 🎁"
   - Optional disabled "Other Events" button
   - Encourages returning without building new features yet

8. **Visual / UX Style**
   - Colors:
     - Primary: #FF6B6B
     - Background: #F7F7F7 → #E0E0F7 gradient
     - Text: #1E1E2F headings, #555555 body
   - Typography:
     - Headline: Bold sans-serif, 32px
     - Body: Medium sans-serif, 18px
   - Cards: Rounded 12px, soft shadow
   - Buttons: Rounded, hover scale 1.05x
   - Mobile-first responsive layout
   - Optional subtle gift icons / emojis

9. **Database Structure (Supabase)**
   - users: id, email, created_at
   - questions: id, sender_id, receiver_email, event_type, reveal_option, created_at
   - wishes: id, question_id, item_name, item_link, price
   - statuses: id, question_id, status_type, timestamp
   - notifications: id, type, receiver_email, content, sent_at

10. **Behavior / Business Logic**
    - Max 3 wishes per receiver
    - Optional reveal after purchase
    - Anonymous delivery
    - Status updates trigger notifications
    - Fast, minimal friction, emotional U.S. audience
    - Designed for TikTok virality

**End of prompt.**
